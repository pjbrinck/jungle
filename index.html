What we're building

A modern approach to provisioning

Today's Infrastructure as Code tooling has grown into a tangle of state files, brittle modules, and ever-growing complexity. We're building a fundamentally better way to describe, evolve, and operate cloud infrastructure — designed from the ground up for the scale, safety, and developer experience that large enterprise engineering organizations demand.

Our product — code-named the MACHINE — is currently in R&D.

Key features of the MACHINE are:

Collaboration

Lets stakeholders — application and service owners as well as infrastructure and security architects — each contribute their own concerns to compound application architectures.

Resource sharing

A structured approach to optimizing resource usage across stages and projects.

Resource relocation

Migrate stateful resources across regions and providers.

Dependency analysis

Lets operators trace the impact and root causes of resource failures.

Developer extensibility

Lets partners and customers build and resell configurable resource producers, adapters to infrastructure providers and assemblers composing other producers — the gears of the MACHINE.
